为什么我选择 Next.js 构建个人网站

2026年7月1日
#前端#Next.js#技术选型

决策背景

在决定搭建 HSS Digital Garden 时，我面临一个关键选择：用什么技术栈？我的核心需求是：

  1. 内容优先，Markdown 编写
  2. 快速上线，不要过度工程化
  3. 静态部署，无需服务器
  4. 为未来的扩展留有余地

候选方案

我考虑了三个方案：纯静态 HTML、Hugo、Next.js 静态导出。

为什么是 Next.js

  • 组件化开发：JSX 和 React 的组件模型让 UI 复用变得自然
  • 静态导出：output: 'export' 配置一行搞定，部署到任何静态托管
  • 渐进增强：V2 需要搜索时，可以无缝切换到 SSR
  • 生态丰富：TypeScript、Tailwind CSS、MDX 支持一应俱全

快速启动 vs 过度工程化

V1 的目标是快速上线。我特意避免了数据库、CMS、API 路由等复杂组件。所有内容就是 Markdown 文件，干净、可控、可迁移。
